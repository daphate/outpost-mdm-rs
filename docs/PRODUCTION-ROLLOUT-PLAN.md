# Outpost — план вывода в production

> **Аудитория:** внутренний инвестор (Frontier Capital).
> **Цель документа:** объяснить состав предложения, которое мы готовы дать
> внешнему заказчику (МО / силовые / частные ЧВК), и перечислить мероприятия
> 24-месячного цикла до полного production. Финансовые оценки **не включены** —
> требуют отдельной проработки CFO + переговоров с конкретными поставщиками.
>
> **Версия:** 0.1, 2026-05-21. Author: Sophia.

---

## 1. Что мы предлагаем заказчику

**Outpost** — комплексное решение для оснащения тактических подразделений
персональными ИИ-ассистентами:

| Компонент | Что входит |
|---|---|
| **Аппаратная часть** | OEM-телефоны (rugged-grade, IP68/MIL-810, перчатко-friendly экран) с предустановленным custom ROM + bundled моделями. Поставка партиями. |
| **Клиентское ПО** | Outpost-Android («Штаб»): on-device LLM (Soldier V25 fine-tune) + VLM (распознавание камерой) + STT (Whisper) + TTS (Piper) + RAG-поиск по корпусу. Offline-first. |
| **Корпус знаний** | Тематические базы знаний (knowledge.db) под конкретные ВУС: медицина, тактика, связь, инженерия, БПЛА, мины, ВКС, военный английский. Distributed как ZIP-архивы через MDM. |
| **Сервер управления** | outpost-mdm-rs (Rust) — управление парком, push-конфигурации, телеметрия, шифрование передачи данных, attestation. |
| **Сопровождение** | Подписочная модель: hotline, обновления моделей, RMA, расширение корпусов под новые задачи. |

**Конкурентные преимущества vs Mobile-LLM конкурентов (e.g. PocketPilot, AI4Soldier):**
- Полностью **offline-ready** (тактический сценарий без связи).
- **On-device privacy** (профили оружия / DOPE-карты / медицинские профили не покидают устройство в незашифрованном виде).
- **Fine-tune под русскоязычный военный domain** (V25 Soldier на собственном корпусе).
- **Российская инфраструктура** (Cloud.ru mirror'а, отечественные mirror'ы; RBAC / device certs на нашей стороне, не через Google Play).

## 2. Текущая зрелость (что готово на 2026-05-21)

- **Server v0.18.17** — production deploy на mdm.secondf8n.tech. Endpoints: enrollment, push, OTLP-телеметрия, encrypted file distribution, ballistics (за feature flag — ждёт crypto review). Admin Web UI на русском, адаптивный.
- **Client rc43-b27** — Soldier V25 LLM + Qwen2-VL camera + Whisper STT + TTS + 26 тематических ZIP-архивов. Tier-aware bundle selection (T0/T1/T2 устройства).
- **Контракты** (working agreements между командами): BALLISTICS-MDM, MDM-DEVICE-CONTROL, CONTENT-DISTRIBUTION, LIBRARY-ARCHIVES, OTEL.
- **Инфраструктура распространения**: Cloud.ru + R2 mirror'ы, byte-identical, HEAD-verified.
- **Pilot devices**: 6 enrolled, регулярно sync-аются, телеметрия в Grafana, panels Fleet/Drill-down работают.

**Готовность к production-deploy конкретному заказчику** на 2026-05-21: ~60-70%.
Что блокирует — раздел 3 ниже.

## 3. План мероприятий — 24 месяца, 7 направлений

Каждое направление — стрим работ. Milestones M1 (3 мес) / M6 (6 мес) /
M12 (12 мес) / M24 (24 мес).

### 3.1 Доработка приложения под требования заказчика

| Срок | Что | Ключевые потребности |
|---|---|---|
| M3 | Сбор ТЗ от первого заказчика; gap-анализ vs текущий rc43 | Военные консультанты доступны для интервью |
| M6 | UI/UX под специфику задач заказчика (кнопки под перчатки, screen brightness control, kiosk-mode) | UX-дизайнер с tactical experience |
| M12 | Дополнительные модальности (если ТЗ потребует): voice-only mode, AR-overlay через USB-камеру/glasses | Hardware partner для AR |
| M24 | Multi-language UI (украинский, английский, китайский для экспорта) | Локализаторы |

**Лаборатория:** оборудование для обучения нейросетей (GPU-cluster — A100 ×4
минимум для fine-tune ≤9B моделей), LoRa-сети для проверки командной связи,
обмундирование/макеты оружия для photo-dataset'ов, 3D-принтер для prototyping
аксессуаров (крепления к разгрузке), полевая экипировка для тестировщиков.

**Команда разработки:** Kotlin/Compose (1-2 senior), Rust backend (1
senior, есть), ML/NLP engineer (1-2), AR/VLM engineer (1), QA tactical (1).

### 3.2 Корпуса данных RAG + обучение LLM на ВУС

| Срок | Что | Ключевые потребности |
|---|---|---|
| M3 | Аудит существующих 26 архивов; gap-анализ под ВУС заказчика | Военный консультант + методолог по контенту |
| M6 | Сбор / адаптация / приобретение корпусов: уставы, тактические наставления, медицинские протоколы, инженерные руководства | Юр.очистка прав на материалы; контракты с правообладателями |
| M12 | LoRA/DPO fine-tune Soldier V25 → V26 → V27 на расширенном корпусе; отдельные fine-tune'ы под ВУС (медик / связист / сапёр / БПЛА-оператор) | GPU-cluster availability; разметчики (ветераны БД, действующие военные мед/спецы) |
| M24 | RAG evaluation framework (golden-set вопросов от заказчика, регрессия моделей) | Постоянный поток обновлений / новых регламентов |

**Военные консультанты и медики, ветераны БД:** ключевой actively-paid resource.
Без них корпус — генерик, ценность для заказчика низкая. Требуется
post-traumatic-friendly engagement model + NDA (некоторые материалы из ВУС
ограничены доступом).

### 3.3 Доработка системы управления парком (MDM)

| Срок | Что | Ключевые потребности |
|---|---|---|
| M3 | Дополнительные admin-функции под workflow заказчика (массовый импорт, отчётность, ролевые модели его иерархии) | Spec от заказчика |
| M6 | On-premise deploy option (заказчик не хочет SaaS на mdm.secondf8n.tech — отдельный droplet в защищённой среде) | DevOps; интеграция с заказчической AD/LDAP |
| M12 | High-availability deploy (2+ replicas, automatic failover) | SRE; репликация SQLite → PostgreSQL миграция |
| M24 | Multi-region (если есть филиалы / зоны / театры действий) | Tailscale-based mesh; региональные mirrors |

### 3.4 Доработка системы криптографической защиты

| Срок | Что | Ключевые потребности |
|---|---|---|
| M3 | Завершить ballistics crypto review (M7 per BALLISTICS-IMPLEMENTATION-SPEC) → production gating | Crypto-engineer (есть в AR Hud team) |
| M6 | Получить лицензии ФСБ: разработка СКЗИ + распространение СКЗИ. Договор с одним из лицензированных центров (ФСТЭК-лаборатория) | Юрист по лицензированию; уставная подготовка; помещения соответствующие |
| M12 | Сертификация СКЗИ в ФСБ (если применяется к шифрованию ВУС-данных) ИЛИ переход на готовый сертифицированный модуль (e.g. КриптоПро) для transport-layer | Решение архитектурное: certify our own vs integrate certified |
| M24 | По необходимости — допуск к гостайне (для подразделений, работающих с СОВ.СЕКР.). Допуски сотрудников, помещения, режимный отдел | Многомесячная процедура — заранее начинать |

**Лицензии ФСБ:** обязательны если шифрование под СКЗИ (даже если используем
известные алгоритмы — сам факт распространения средств шифрования
лицензируется по ПП РФ 313). Стартовать процесс **до** реальной production
поставки.

**ФСТЭК (защита информации):** параллельный трек. Регулирует не шифрование,
а в целом защиту информации (включая КИИ, если заказчик — субъект КИИ).
Сертификация по требованиям ФСТЭК — отдельный путь.

## 4. Что нужно от инвестора

### 4.1 Команда (приоритеты найма / партнёрства)

В порядке убывания критичности:

1. **Military domain consultants** (ветераны БД, действующие военные медики/связисты/сапёры) — без них корпус low-value.
2. **Crypto-engineer** для server-side review + СКЗИ-сертификация. У AR Hud есть, но full-time дедикейтид для production может потребоваться.
3. **Юрист по лицензированию** (ФСБ / ФСТЭК / ПД). Не разовый — постоянная работа.
4. **DevOps / SRE** для эксплуатации (24/7 on-call).
5. **ML/NLP engineer** для регулярных fine-tune циклов (1 раз в 2-3 месяца).
6. **QA tactical** (с полевым опытом — иначе test cases оторваны от реальности).
7. **Customer Success Manager** для прямого engagement с заказчиком.

### 4.2 Лаборатория

- GPU-cluster: ≥4 A100 (или эквивалент H100 / отечественный аналог если доступен) для fine-tune Q4_K_M моделей до 9B.
- Тестовые устройства: ≥20 OEM-телефонов разных tier'ов (T0/T1/T2).
- LoRa-станции: ≥6 точек для покрытия полигона.
- Полевая экипировка: разгрузки, наколенники, перчатки разных типов (зимние/летние/тактические) для UX-теста.
- 3D-принтер промышленный (FDM 300×300 минимум) для аксессуаров.

### 4.3 Юридические / регуляторные процессы

- **Лицензия ФСБ** на разработку СКЗИ (~6 мес procedure).
- **Лицензия ФСБ** на распространение СКЗИ.
- **Аттестация по требованиям ФСТЭК** (если КИИ-объект или ПДн уровня УЗ-1/2).
- **ПП РФ 1119 / ФЗ-152** — модель защиты ПДн (даже если ВУС-данные не ПДн в строгом смысле, всё равно безопаснее compliant).
- **Реестр отечественного ПО** (ПП РФ 1236) — для product Outpost (даёт преференции при госзакупках).
- **Если гостайна** — допуск 3-й формы для сотрудников работающих с грифом, режимный отдел, СЗИ-аттестация помещений.

### 4.4 Партнёры

- **OEM-завод**: контракт на стабильные поставки rugged-телефонов с возможностью firmware customization. Кандидаты — Ulefone (текущий tier-2), Honor, Realme rugged-line, отечественные OEM (Aquarius / Yadro если они делают мобильные).
- **Сертификационная лаборатория ФСТЭК** для предъявления на аттестацию.
- **Cloud-hoster** для production deploy (Cloud.ru — РФ, Selectel, Yandex Cloud).
- **Mil-grade testing lab** — климатические / вибро / EMC испытания.

### 4.5 Сертификация / стандарты

- **ОТТ ВТ** (общие технические требования к военной технике) если будем classified as such.
- **ГОСТ Р 56939-2016** (требования к безопасной разработке ПО).
- **ГОСТ Р 51275** (безопасность объектов информатизации).
- **ISO 27001** (опционально, для коммерческих заказчиков).

## 5. Риски и mitigation

| Риск | Mitigation |
|---|---|
| **OEM поставщик меняет железо** между партиями (другой SoC, RAM) — модели могут не помещаться | Спецификация tier requirements в контракте; QA на каждую партию |
| **Изменение НПА** (новые требования ФСТЭК / запрет западных primitives) | Тренировать crypto-engineer на регулярный обзор законодательства; alternative модули заранее |
| **Утечка моделей конкуренту** (Soldier V25 файлы попали в открытый доступ) | DEK-обфускация на устройстве + remote-wipe + per-device key release |
| **Военный консультант недоступен** (тяжелая обстановка СВО) | Запас relationship'ов; backup консультанты; контракт на минимальное участие |
| **Зависимость от Cloud.ru / R2** | Multi-mirror архитектура уже есть; добавить третий российский mirror |
| **OPSEC компрометация** (через утечку метаданных, кто куда сходил) | Tailscale-only deploy; minimum logging; обучение пользователей |

## 6. Что считается «production-ready» (acceptance criteria для инвестора)

Готовность к **first paid customer delivery** — outcome контрольных точек:

- [ ] Лицензия ФСБ на распространение СКЗИ получена.
- [ ] Crypto review ballistics + остальных flow подписан AR Hud crypto-engineer'ом.
- [ ] ≥1 пилотное подразделение использует Outpost минимум 3 месяца без critical incidents.
- [ ] OEM поставка ≥50 устройств за один заказ возможна (контракт с поставщиком).
- [ ] Hotline и RMA work как обещано в SLA (response ≤24ч).
- [ ] Модель монетизации согласована (per device + monthly subscription / per seat / per contract).
- [ ] Корпус знаний для ≥3 ВУС готов и evaluation passed.
