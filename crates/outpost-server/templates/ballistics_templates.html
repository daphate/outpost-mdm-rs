Как это работает

  • Template — plaintext шаблон оружия (или патрона) от командования. By design shareable content (admin осознанно публикует) — хранится в БД без encryption.
  • Personal user records (создаваемые на устройстве) — encrypted-at-rest с per-record DEK. Server opaque (zero-knowledge envelope).
  • Когда client получает template (GET /api/v1/ballistics/admin/templates) и user accept'ит — client локально encrypt'ит и POST'ит в /api/v1/ballistics/<kind> как обычный personal record.
  • Retract — soft-cancel. Устройства, ещё не получившие/accept'нувшие, скроют suggestion. Уже accept'нутые остаются (personal record уже у user'а).
