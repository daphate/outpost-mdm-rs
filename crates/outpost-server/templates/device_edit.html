Push новой версии приложения

Управление APK-обновлениями. Есть три способа доставить новую версию на это устройство:

  1. Pin версии в форме сверху («Закрепить версию») — устройство получит её на ближайшем /sync в поле update_available; AR Hud покажет пользователю кнопку «Обновить». Самый мягкий способ.
  2. Раскатка через rollouts на /applications — canary в группе → fleet-wide. Та же механика update_available, но автоматически по политике.
  3. Push install-apk — командой ниже. Устройство получит команду в commands[] и (когда AR Hud client это реализует, см. MDM-DEVICE-CONTROL-CONTRACT §3.4) скачает и установит APK. Без silent-install DPM — будет user-prompt от Android PackageInstaller.

rotate-cloudru-creds

Хотя бы одно поле обязательно. Партиал-ротация (только secret) поддерживается. Поле передаётся в client'ский ModelPreferences.setCloudruCreds через push command.

Soft revoke: client применяет на ближайшем /sync. Текущий device_token остаётся валидным пока клиент не уберёт его сам. Для hard revoke (немедленный 401 на всех endpoint'ах) используй «Удалить устройство» ниже.

Опасная зона — hard revoke
