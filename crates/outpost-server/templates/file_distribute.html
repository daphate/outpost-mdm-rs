Как работает encrypted distribution

  • Файл шифруется AES-256-GCM один раз, под случайным DEK.
  • Для каждого получателя DEK wrap'ится через ECDH(eph_priv, recipient_pubkey) → HKDF → AES-GCM. Один blob, N оберток.
  • Только устройства с reported pubkey'ем (после enroll'а rc42 b37+) могут получить — остальные пропускаются.
  • Push-команда `fetch-encrypted-file` уходит через /sync (≤30 мин), клиент скачивает ciphertext, декодит, install'ит по target storage.
