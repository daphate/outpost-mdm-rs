Как это работает

  • canary — раскатка идёт только устройствам выбранной группы. Monitor каждые 60с проверяет crash-rate; если > gate — auto-rollback.
  • fleet — раскатка всем устройствам клиента. Auto-промоушн из canary происходит при достижении canary_until_at + clean crash-rate.
  • paused — приостановлено admin'ом. /sync не отдаёт update_available для этой роллаут-строки. Можно вернуть в canary/fleet.
  • rolled_back — окончательно остановлено. Audit-trail (причина + время) сохраняется. Чтобы повторить — создать новую раскатку.
  • Per-device pin на странице устройства имеет приоритет над любой роллаут-policy.
